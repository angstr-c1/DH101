Week 1 – Reverse Engineering

The Artifact

Make 1: Orbis: The Stanford Geospatial Network Model of the Roman World

https://orbis.stanford.edu

1. What is Being Made?

This project is an interactive map of the Roman World during roughly 200 CE. It shows different trade routes and different modes of transportation for each. By doing this, it aims to create an experience in which a regular map can’t: allowing the user to simulate how difficult travel was during certain conditions like weather or transportation speeds.

2. What Is the Project Made From? (Data)

The main form of “data” in this project is the 632 sites that represent locations like urban centers or sea ports. On top of this, “time” would also be a form of data in this instance. However data like distance, while gathered from reputable sources like the National Imagery and Mapping Agency or geographical references. However, it seems to lack exact historical data of, for instance, a traveler writing in a letter about the length of his journey from Rome to Constantinople by sea, and rather focuses more on simulating these travels through data obtained in the modern day.

3. Tools, Algorithms, or Systems

This project uses tools like GIS, however, most of the actual plotting and software seems to be entirely done by scratch. I could not find any mention of AI, and I don’t believe that any AI was used in the creation of this project. Especially considering that, although not all of the data was compiled by the same people, it was still compiled and collected by people.

4. Human Labor and Decisions

This project seems to be entirely done by humans. Because of this, I feel as though there was some human judgment in the designing process. Especially with how some of the routes are defined. Obviously, a human can only look through so many sources in one’s life, so I believe that is why they went with the simulation of time rather than looking for if there was any historical information available.

Angstrom

5. Design as Argument

This project does its job very well. By having this as an interactive map, it allows for the viewer to get a “sense” of how it would have been to travel in the Roman Empire. This is especially pertinent in regards to American viewers because most of us haven’t been to the Mediterranean to even get an idea of how large the area is. The main downside, however, is that it doesn’t give the “human” side of history. You aren’t going to learn about any individual stories but rather the broad idea.

Process Notes

This project was built as a close reading of Orbis, focusing on what the interface communicates through data, structure, and interaction. I looked at how the map translates travel into a visual simulation, and how the design choices shape the user’s sense of scale and difficulty.

The main decisions I made were about what details to emphasize: the role of data, the use of GIS, and the way the project turns historical information into an interactive argument. I also noted that the project did not appear to rely on AI, which helped frame it as a human-made digital system rather than an automated one.

Reflection

In this project, “making” does not directly mean physically creating something, but rather designing and creating an interactive digital map. Because of this, I would still say it follows a more traditional idea of making, whereas although it wasn’t physically created, it was created by humans in a digital space using data collected by other humans. For instance, the algorithms used to determine the time of travel by sea are created entirely by people for the sole purpose of simulating the time it would take to travel such a distance by using both historical and current data of winds, currents, and navigational capabilities. As a result, I believe that this is still “making” because everything was made for the purpose of creating something new. For instance, if these people were only compiling other data and ripping off other researchers without much extra thought, I would not constitute it as “making” because there was minimal or no effort put behind it to create this new thing.

Something that I find unique about this project is that there wasn’t any (to my knowledge) use of artificial intelligence. Because of when it was published, there was much more limited access to artificial intelligence. However, the creators still use “machines” in order to create their project. I believe, however, that because all of the “thinking” was done by people, the machines, like GIS, would be viewed as a tool rather than something that did the work for them. However, if this was performed by AI, I would be more hesitant to count it as “making” because of the lack of human input in the actual development of the project. (Unless you would consider prompting AI as developing, which I am still unsure on)
